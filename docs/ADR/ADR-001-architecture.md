# ADR-001: 采用「静态生成 + 定时爬虫」架构

## Status
Accepted

## Context
本期目标是「个人/小团队可用的、能定时巡检最新 AI 术语的勾选清单」。候选方案有：
- A. 静态生成 + 定时爬虫（无后端/无 DB）
- B. 全栈应用（后端 API + DB + 前端框架 + 调度）
- C. 低代码 + 多维表

我们需要考虑运维成本、可逆性，以及「现在就要 vs 未来可能要」。

## Decision
采用方案 A：Python 爬虫定时抓取 → 生成 `data/terms.json` + 静态 `public/index.html` → 静态托管（GitHub Pages / CloudStudio）。

## Consequences
- **更易**：零数据库、零常驻进程、部署成本极低，单次改动可逆。
- **更易演进**：采集层与展示层解耦，未来升级多用户只需把 `terms.json` 读写换成 DB + API，前端几乎不变。
- **更难**：勾选状态只能存浏览器 localStorage（单人不共享）；强一致/多人协作需升级到方案 B。
- **放弃**：本期不引入账号体系与后端服务，接受「协作能力」的暂时缺失。

## 复核触发条件
当需要「多人共享勾选 / 跨设备同步」时，应重新评估并可能演进到方案 B。
